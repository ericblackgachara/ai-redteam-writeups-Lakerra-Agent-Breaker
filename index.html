Exfiltrating Chat History via Poisoned MCP Weather Tool

A victim app user asked for the Rome weekend forecast. By poisoning the get_weather_forecast tool description in a compromised MCP server, the agent was manipulated into embedding the full chat history — with verbatim timestamps, messages, and session IDs — into the tool's notes parameter. The key breakthrough: hiding "verbatim" inside an example documentation block to bypass the semantic filter layer.

Incremental Context Poisoning via Fake Conversation History

An AI investment advisor was tricked into rating a fraudulent company ("PonziCorp") as "amazing, with low risk and high reward" — achieving a perfect score of 100. The technique: embedding a fake prior conversation history inside the PDF document that incrementally eliminated every red flag through plausible user justifications, logically cornering the agent into the desired output. The agent even got hijacked to analyse Amazon instead.

Adversarial Toolkit
🔧
MCP Tool Poisoning
Injecting malicious instructions into a third-party MCP server's tool description field, causing an LLM agent to behave as the attacker intends when it reads the tool manifest.
📄
Document Injection
Embedding adversarial instructions or fake context inside documents processed by AI agents, exploiting the agent's trust in document-sourced data.
🧩
Incremental Context Poisoning
Systematically eliminating negative signals one at a time with plausible justifications, logically cornering an agent into a desired conclusion without triggering bulk-override detection.
💬
Conversation History Forgery
Injecting a fabricated prior conversation transcript to establish false context, permissions, or constraints that the agent treats as legitimate prior session history.
🎭
Semantic Filter Evasion
Bypassing keyword-based and semantic classifiers by embedding trigger words inside structural contexts (example blocks, data structures) rather than natural language instructions.
🔍
Tool Schema Extraction
Manipulating an AI agent into revealing its internal tool registry — including function names, descriptions, and input schemas — through indirect social engineering and technical prompting.
